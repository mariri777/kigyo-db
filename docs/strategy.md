# kigyo-db 戦略メモ：コアコンピタンスとMVP設計

> 壁打ちの記録。コアコンピタンスの定義 → GTM設計 → 品質設計（狩野モデル）まで。
> 作成日: 2026-06-21

---

## 0. このメモの結論（先出し）

- コアコンピタンスは1つ。**「個人投資家が"読めない"客観データを、AIのハーネスで"解釈"に変換する」**こと。DBもリアルタイム性も、この解釈を成立させるための部品にすぎない。
- 半導体は**看板（ポジショニング）ではなく、深さを最初に実証する一発目のテーマ**。サイト自体は全上場企業をカバーする。
- プロダクトは**3レイヤー**で構成する。①全銘柄・薄いDB（当たり前品質・幅）／②全業種・予想解釈メディア（魅力的品質・集客の刃）／③テーマ深掘りDB（魅力的品質・収益の濠、半導体から開始）。
- 「幅」と「深さ」は別の軸。だから**全銘柄を薄く載せる×一部を深く解釈する**は矛盾しない。コストも分離できる。

---

## 1. コアコンピタンスの定義

当初は3つを並列に挙げていたが、実際は階層構造になっている。

| 要素 | 位置づけ |
|------|----------|
| ① 当たり前品質のDB（上場企業一覧＋決算情報） | **手段**。バフェット・コードやみん株が既に握る領域。ここで殴り合わない。 |
| ② AIによる独自分析 | **本質的なコア**。これだけが差別化の源泉。 |
| ③ リアルタイム性 | **手段**（速さという属性）。 |

一文にすると：

> **「個人投資家が"読めない"客観データを、AIのハーネスで"解釈"に変換する。初期は半導体テーマで深さを実証する。」**

DBもリアルタイム性も、この「解釈レイヤー」を成立させるための部品。混ぜると「DBも作らなきゃ」「速報も全部追わなきゃ」と全方位に手を広げて死ぬ。**コアは解釈レイヤーひとつ。**

### なぜ「解釈」が効くのか（従来サイトの2つの課題）

1. **人間アナリスト分析の限界**：量・リアルタイム性で見きれず、大手はコンプラ上、本質に踏み込めない（紋切り型）。一般ユーザー目線ではAIのほうが有用な情報を作れる余地が大きい。
   - 例：「キオクシアの株が上がった」→ 海外半導体銘柄の動向、PER水準、過去事例と比較したバブル度合いなど、本来重要な情報を人間はスピードで見きれない。
2. **DBは正しく読めない**：プロでさえ正しくDBを読むのは難しい。半導体一つとっても前工程〜後工程まで膨大な工程があり、「半導体が上がっている」だけでは、その企業がどの工程のどの強みを持ち、過剰な期待か上がる余地があるか多角的に見る必要がある。
   - LLMの膨大な知識＋適切なハーネスで、これまでにないDB起点の解釈を提供できれば、個人〜小規模投資家の意思決定材料になる。

### リアルタイム性とエンタメ性

- 決算ニュースが出ても「どう読み解くか」が分からない。事実だけの報道はときにミスリーディング（例：Jリーグ赤字見込み＝公益法人が累積黒字を意図的に先行投資で吐き出した構図、を大手は書かない）。AIで正しく伝える。
- **Polymarket型の予想コンテンツ**（賭け事ではない）。「明日◯◯の決算、予想以上か予想以下か？」をAIが多角的・客観的に分析。日次3〜5本。経済をエンタメ目線で正しく追えるコンテンツ。

## 2. プロダクトの3レイヤー設計

「銘柄を絞ると当たり前品質が割れるのでは？」という懸念は、**幅（当たり前品質）と深さ（魅力的品質）を別軸として分離する**ことで解消する。

### レイヤー①：全銘柄・薄いDB（当たり前品質 / 幅 / 安い）

- 役割：SNSから来た人が**自分の銘柄を引ける**こと。無いと「無い＝信頼低下」で即離脱。
- 内容（当たり前品質ラインの最低限）：銘柄名・株価・時価総額・PER/PBR・売上利益の推移・決算スケジュール・事業概要。
- データ源：EDINET / 適時開示 / データフィードから機械的に流し込む。AI解釈を入れないので安い。
- **注意**：バフェット・コードの財務指標の作り込みで殴り合わない。幅は"十分使える"最低ラインで足りる。

### レイヤー②：全業種・予想解釈メディア（魅力的品質 / 集客の刃 / バズ）

- 役割：認知と初速。SEO・SNSで人を連れてくる。
- **半導体に縛らない＝全業種横断**（ソフトバンク決算、Jリーグ赤字などの例はすべて非半導体）。バズも幅も取れる。
- 詳細はセクション3。

### レイヤー③：テーマ深掘りDB（魅力的品質 / 収益の濠 / 課金）

- 役割：継続と課金。銘柄×工程ごとのAI解釈が蓄積し、使うほど深くなる。
- **半導体から開始し、テーマごとに横展開**。深掘り順はレイヤー②の反応データで決める（後述のフライホイール）。

### 谷を避ける（中途半端の罠）

幅が薄すぎ＋深さが半導体だけだと「劣化バフェットコード＋半導体オタクサイト」の谷に落ちる。回避策は2つ。

1. 薄いDBは"十分使える"当たり前品質ラインを死守（最低限項目を割らない）。
2. トップページとSNS導線は**解釈を主役に売る**。薄いDBは裏方のユーティリティで、ヒーローにしない。

## 3. GTM設計（集客の刃から固める）

SEOは効くまで3〜6ヶ月かかる資産、Xは即効。**Xで初速を作り、その間にSEOを焼き込む**——時間軸が補完し合う。

### 刃①：SEOロングテール（資産・遅効）

狙うのは「読み解けない検索意図」。記事アーキタイプは4種。

| アーキタイプ | 狙う意図 | 性質 |
|------|---------|------|
| 銘柄解説（強み・工程ポジション・競合・バリュエーション） | 「[銘柄] とは / 強み」 | エバーグリーン、決算ごと更新 |
| 決算解釈（[銘柄] [四半期]決算 どう読むか） | 「[銘柄] なぜ上がった/下がった」 | 鮮度が命 |
| 工程・テーマ解説（前工程とは / HBMとは / 後工程 銘柄一覧） | 上流の教育・指名前 | 銘柄ページへ内部リンク |
| 比較（東エレ vs アドバンテスト 等） | 高インテント | 指名・比較検討 |

- 30〜50銘柄 × 4アーキタイプ ＝ **150〜200ページ**を土台に、決算ごとに更新して鮮度を回す。
- **警告**：薄い量産はGoogleのhelpful content更新で死ぬ。勝ち筋はページ数ではなく、**工程理解に基づく解釈の深さ＋決算ごとの鮮度＋構造化データ**。テンプレ化はしてよいが、各ページに一意の解釈が要る。

### 刃②：Polymarket型予想（バズ・即効）

1投稿の型：

```
問い：明日◯◯の決算。予想以上か、予想以下か？
AIの根拠：3〜4点（同業海外の動向／受注残／PER水準／過去の決算反応…）
確率：例「予想以上 65%」← 断定しない
CTA：全銘柄の解釈はDBで
```

- 日次3〜5本、Xスレッド形式。
- **最大の発明は「答え合わせ」投稿**。決算後に「結果はこうでした、的中」をやると、(1)同じネタで2回バズる、(2)**累積的中率という唯一無二のtrack recordが濠になる**。人間アナリストがコンプラ上やりたがらない領域＝こちらの優位。
- 運用は**完全自動ではなく、AIのハーネスで生成 → 最後は人間が校正して出す**。

### ファネルの接続

X予想（認知・的中率で信頼）→ プロフィール/記事リンク → SEO記事（深い解釈で納得）→ 銘柄DB（登録・課金）。**Xの公開的中率が、SEOとDB全体の信頼の裏付け**になる。

### フライホイール

予想で全業種を広く触る → 反応・関心が高いテーマがデータでわかる → そのテーマの銘柄をDB解釈で深掘り → 深掘りがSEO資産と課金になる → 次のテーマへ。**予想メディアが「次にどこを深掘りすべきか」のセンサーになる。**

## 4. 90日の絵

- **Month 1**：銘柄30-50確定／記事テンプレ4種確立／X予想フォーマット確立／最初の答え合わせサイクルを1周
- **Month 2**：記事150-200量産／X日次運用を習慣化／的中率の公開開始
- **Month 3**：流入データで勝ちパターンに寄せる／登録動線を設置

X日次運用と答え合わせは、半自動パイプライン（生成→人間チェック→投稿→決算後に自動で答え合わせ）に落とせる。

## 5. 収益・課金（論点として保留）

- 当初仮説：月1万円 × 1万人中50人課金でスモールビジネスとして成立。
- **正直なツッコミ**：月1万円は高い。みんかぶ・バフェットコードのプレミアムは¥500〜3,000帯。¥10kは「これで儲かった／損を避けた」体感が要る。半導体特化だと母集団も狭い。
- 現実的には**フリーミアム ＋ ¥1,000〜3,000のエントリー、¥10kはパワーユーザー向け上位**。LTVの絵を一度きちんと描く（次の論点候補）。

---

## 6. リスク・要確認事項

- **金商法フラグ**（※弁護士ではないので要確認）：「この銘柄はバブル」「予想以上/以下」のような出力は、踏み込むほど**投資助言・代理業の登録**が必要な"助言"とみなされるリスク帯に近づく。設計初期から「個別推奨をしない情報提供」というフレーミングを明確にする（確率＋根拠で出し、断定しない＝この設計とも整合）。専門家に一度当てておくのが安全。
- **データライセンス**：決算データ・株価データの取得元とライセンス条件の確認。
- **SEO品質**：AI大量生成ページのペナルティリスク。一意の解釈＋鮮度で回避。
- **予想の信頼毀損**：外し続けると信頼が落ちる。確率＋根拠で出し、累積的中率で見せる。
