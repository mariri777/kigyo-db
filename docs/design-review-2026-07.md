# 超!企業DB デザインレビュー(2026-07-02)

> 情報設計・UX・ビジュアル・アクセシビリティ・パフォーマンス・SEO・先進性の 7 観点で、
> 実ブラウザ確認(デスクトップ 1440px / モバイル 390px)+ 全公開ページのコード精査を実施。
> 戦略文書 `docs/strategy.md` との整合も評価。改善の優先順位は末尾。

---

## 総評

**土台は平均以上、しかし「信頼」を削る細部が最大の弱点。**

デザインシステム(モノクロ + emerald、shadcn 準拠トークン、日本語タイポグラフィ)と銘柄詳細ページの情報密度は、個人開発の域を超えた水準にある。一方で、(1) データ鮮度と表示の不整合、(2) トップページ下半分の「準備中」の連続、(3) キーボードフォーカスの体系的欠落、(4) SEO 戦略の中核(sitemap / 構造化データ)の不在——という「信頼と発見可能性」に直結する欠陥が、戦略(解釈で信頼を勝ち取る)の足を引っ張っている。

## 現状の強み(保持・強化すべき資産)

- **銘柄詳細ページの構成力**: 「この会社をひとことで」→ アナリスト目標 → 市場評価 → 配当 → テクニカル → 決算 → 業界ポジショニング → 株主 → 100年紙芝居、という語り口は競合(バフェット・コード等)にない「解釈ファースト」の体験。戦略レイヤー③の器としてすでに強い。
- **日本語タイポグラフィ**: `palt` / `line-break: strict` / `word-break: auto-phrase` / `text-wrap: balance·pretty`(globals.css)は日本語 Web では最高水準の配慮。
- **数値の扱い**: `tabular-nums` + JetBrains Mono の徹底で、金融データの可読性が高い。
- **ダークヒーロー(ホーム市場サマリ / 銘柄詳細 / 予測)**: ライト基調に対する視線誘導として機能している。
- **検索の実装品質**: 200ms debounce + AbortController + combobox ARIA + キーボードナビ(_SearchBox.tsx)。エッジキャッシュ付き API も適切。
- **記事詳細のエディトリアルレイアウト**: TOC 追従、KPI カード、インライン銘柄カード、「紐づく企業」サイドバー。
- **画像パイプライン**: next/image 経由で最適化・遅延読み込みが機能。

## 課題(観点別)

### 1. 信頼性の毀損 — 最優先で潰すべき

金融情報サイトの通貨は信頼。現状、以下が体験の中で信頼を削っている:

| 問題 | 場所 |
|---|---|
| 「株価 · リアルタイム」「LIVE」バッジ(実データは日次スナップショット) | `stocks/7203/_renderer.tsx:130` |
| データが 6/28 のまま(閲覧日 7/2)でも鮮度警告なし。「本日の市場サマリ」と表示 | ホーム全体 |
| 注目テーマが「0.00%」×3 で表示(データ欠損がそのまま露出) | ホーム市場サマリ |
| TOPIX 値「–」なのに上向き矢印アイコン | ホームヒーロー(モバイルで顕著) |
| 予測詳細で「観測中 ・ 解決済」という矛盾ラベル | `forecasts/[id]/page.tsx:218` |
| 画像ミスマッチ: 1867年豊田佐吉のスライドに白いヒューマノイドロボット、トヨタ記事にマクラーレン、ニデック記事に車内の人物 | 紙芝居 / 記事サムネ(Unsplash プレースホルダー) |
| D1 接続失敗時にハードコードのダミー指数・ダミー AI サマリが「本物のように」表示される設計 | `page.tsx:72-92`(INDICES / AI_SUMMARY) |
| 業種チップの銘柄数がハードコード(実 DB とズレる) | `page.tsx:140-153` |

**方向性**: データに `asOf` を必ず持たせ、鮮度をラベルで正直に見せる(「6/28 大引け時点」)。欠損は「–」+ 中立表示に統一し、矢印・% を出さない。ダミーへのフォールバックは開発環境のみに限定。画像は「意味が合わない写真より、意味のある図形・チャート・ソリッド背景」へ全面差し替え。

### 2. 情報設計(IA)

- **トップページの下半分が空洞**: 「今、注目すべき企業/業界」は 3 枚すべて COMING SOON、「AIの明日予想」は空(「まもなく更新されます」)。ファーストビュー以降が看板倒れで、戦略の「解釈を主役に売る」を裏切っている。準備中セクションはトップから外すか 1 枚に圧縮し、実在するコンテンツ(記事・予測アーカイブ・銘柄詳細の見どころ)を前に出すべき。
- **グローバルナビ「注目企業」が準備中セクションへのアンカーリンク**(`/#featured`)。ナビの一等地から Coming Soon に誘導している。
- **「業界」というハブページが存在しない**: パンくずに「自動車・輸送用機器」が出るが、リンク先は実質 `/stocks?sector=` のフィルタのみ。戦略レイヤー②(業界俯瞰)の受け皿がない。
- **フッターが機能していない**: ホットテーマ 8 チップ全部 `href="#"`(_Footer.tsx:39)。サイトマップリンク・運営者情報・プライバシーポリシー・問い合わせがゼロ。メディアとしての E-E-A-T シグナル欠落。
- **予測 → 銘柄の導線が弱い**: 予測は指数(日経・SOX・USD/JPY)のみで、個別銘柄予想(戦略の Polymarket 型・決算予想)がまだ IA 上に存在しない。

### 3. UX ディテール

- **検索に関連度がない**: `stockRepo.search` に ORDER BY がなく(stockRepo.ts:194-212)、「トヨタ」でトヨタ紡織がトヨタ自動車より上に出る。前方一致 > 部分一致、時価総額降順のランキングが必要。プレースホルダーは「銘柄・業界・キーワードを検索」だが、実際は銘柄しか返さない(記事・業界が検索対象外)。「すべての結果を見る」もない。
- **死んだ CTA**: 予測一覧の「解決済」カード 2 枚は「根拠とシナリオを読む」というリンク風テキストがあるのにリンクなし(`<Link>` は観測中の 2 枚のみ)。
- **銘柄一覧の行がクリックできない**: タップ領域は右端の小さな ↗ のみ。行全体をリンクにすべき。
- **一覧フィルタが「適用」ボタン方式**: 選択即反映(+URL 同期)が現代の標準。なお `/stocks` は searchParams 対応済みなので、フロントの UX だけの問題。
- **モバイルの銘柄一覧**: 列がコード/銘柄名/株価の 3 つに縮退し、デフォルトソート基準の時価総額も前日比も見えない。投資家がモバイルで最も見たい「前日比」を優先表示すべき。
- **記事一覧の絞り込みが URL に入らない**: カテゴリ・検索状態が共有・ブックマーク不可。
- **紙芝居(30枚)**: キーボード対応・進捗表示は良いが、1 スライドずつしか進めず 30 枚は長い。章スキップやスクラブができない。スクロール中に白画面が出る場面もあり、重さの兆候。

### 4. アクセシビリティ / モバイル品質 — 体系的な欠落

- **`focus-visible` の使用が全サイトで 0 件**。`focus:outline-none` が 32 箇所。キーボード利用者はフォーカス位置をほぼ視認できない。デザイントークン(`--ring`)は定義済みなのに未使用。
- **`prefers-reduced-motion` 対応が 2 箇所のみ**。`animate-pulse` 7 箇所(ナビの赤ドット、LIVE バッジ等)や多数の transition が無条件で動く。globals.css への一括ルール 1 本で大半を解消可能。
- **スキップリンクなし**(全ページ共通、`<main>` に id もない)。
- **モバイルメニューにフォーカストラップがない**(_Header.tsx: Escape のみ対応)。ダイアログ役割は宣言しているが、Tab が背景に抜ける。
- **タッチ最適化プロパティが 0 件**: `touch-action` / `overscroll-behavior: contain` / `env(safe-area-inset-*)` がリポジトリ全体で不使用。モバイルメニュー・紙芝居・ダイアログでスクロールが背面に抜け、iOS ノッチも非考慮。
- **紙芝居の矢印キーが `window` レベルで奪われる**(_StorySlider.tsx:62)— スライダーにフォーカスがなくてもページ全域の ←→ を乗っ取る。
- **クリックできそうで出来ない UI**: トップのサブ指数カード・注目テーマが `cursor-pointer` 付き `<div>`(リンクなし)、ブックマーク/共有ボタンは `onClick` 自体がない飾り。
- 詳細な file:line 一覧: `docs/design-review-2026-07-code-findings.md`(チケット分解用)。

### 5. ビジュアルデザイン

- **一貫性は高いが、記事タイトルのセリフ体が脆い**: `var(--font-serif, ...)` の `--font-serif` が未定義(articles/[slug]/page.tsx:273)。実際はシステム明朝で描画され、OS によって游明朝/ヒラギノ/Noto Serif とバラつく。エディトリアルの顔になる部分なので、Noto Serif JP(可変)を正式採用して統一すべき。
- **ダークモード未対応なのにシグナルが矛盾**: `viewport.colorScheme: "dark light"`(layout.tsx:99)vs `html { color-scheme: light }`(globals.css:117)。ダーク OS 環境でフォーム部品だけダークになる系の不具合予備軍。金融ツールはダーク需要が強く、トークン基盤はあるので対応価値が高い。
- **COMING SOON カードの淡いグラデーション + 大面積**は、モバイルで 1 画面を占有し「空虚さ」を増幅している。

### 6. パフォーマンス / 技術的負債

- `(main)/page.tsx` **1,357 行**、`_renderer.tsx` 869 行の単一ファイル。セクション単位のコンポーネント分割が改善の前提条件になる。
- **`/stocks/7203` にレンダラーの実体が置かれ、汎用 `[code]` が prototype ディレクトリから import する逆転構造**(`[code]/page.tsx:9-10`)。`_components/stock-detail/` 等へ正規化すべき。
- `unsplashUrl()` ヘルパーが 4 ファイルに重複。
- フォント: Noto Sans JP を `preload: false` で 3 ウェイト全読み(サブセット未活用)。日本語 Web フォントの初期描画コストは要計測。
- 紙芝居 30 枚分の DOM + 画像が銘柄詳細に常駐(遅延はしている)。スクロール中の白画面はこの重さと関係する可能性。

### 7. SEO / 先進性

- **全記事(公開済み含む)が noindex**: `articles/[slug]/page.tsx:44` が無条件で `robots: { index: false, follow: false }` を返す。「記事 150〜200 本で SEO を焼き込む」戦略を根幹から無効化する最重大バグ。draft 隠蔽が目的なら status で分岐すべき。
- **`sitemap.ts` が存在しない**(README には「配置済み」と記載 — ドキュメントと実装の乖離)。3,800 銘柄 × 記事のロングテール SEO 戦略にとって致命的な欠落。
- **`src/lib/seo/structuredData.ts` に BreadcrumbList / CollectionPage 等の JSON-LD ビルダーが実装済みなのに、どこからも import されていない**(デッドコード)。戦略文書は「構造化データが勝ち筋」と明言しているのに未配線。
- **LCP 画像の外部ホストに preconnect なし**(images.unsplash.com / kigyo-assets.cho-super.com)。
- メタデータ・OG/Twitter 画像・robots・manifest は整備済み(良)。
- **先進性の現在地**: チャートは静的 SVG スパークラインのみで、ホバー・期間切替・比較などのインタラクションがない。ウォッチリスト・銘柄比較・スクリーナー・AI への追い質問といった「AI ネイティブ」な操作体験も未着手。ただし「予測に確率 + 根拠 + 答え合わせ」という編集フォーマット自体は独自性が高く、UI の先進性より **track record の可視化(的中率、確率推移チャート)** の方が差別化に効く。

## 戦略(strategy.md)との整合性

| 戦略の主張 | 現状の UI | 評価 |
|---|---|---|
| 解釈を主役に売る。薄い DB はヒーローにしない | ホームは市場サマリ+AI まとめが先頭に来ており方向は正しいが、下半分の空洞と Coming Soon がメッセージを弱める | △ |
| 「答え合わせ」と累積的中率が最大の濠 | アーカイブ 0 件・的中率の表示場所自体が存在しない | ✗(UI の器から作る必要) |
| 全銘柄の当たり前品質を死守 | 一覧・詳細とも実データで機能。ただし欠損表示(0.00%、–)が品質ラインを割る | △ |
| SEO: 構造化データ + 鮮度で勝つ | sitemap なし、JSON-LD 未配線、鮮度表示なし | ✗ |
| 金商法リスク: 断定しない・個別推奨しない | 確率 + 根拠のフォーマット、フッター免責は整合。「ガチで予測」コピーは要法務確認 | ○ |

## 改善ロードマップ(提案)

### P0 — 信頼の即死バグを止める(効果大・工数小)
1. **全記事 noindex の解除**(published のみ index 許可に分岐)— SEO 戦略の前提条件
2. 「リアルタイム/LIVE」表記の撤去 → `asOf` ベースの鮮度ラベルへ(全ページ共通コンポーネント)
3. 欠損値の表示規約を統一(0.00% や上向き矢印を出さない)
4. 死んだ CTA の修正: 解決済み予測カードへのリンク付与、フッターのテーマチップ(実装するか消す)、飾りのブックマーク/共有ボタン(実装するか消す)
5. 検索の関連度順(前方一致優先 + 時価総額順)
6. ミスマッチ画像の排除(紙芝居・記事のプレースホルダーを図版・ソリッドへ)
7. a11y クイック対策: globals.css に reduced-motion 一括ルール + skip link 1 本 + StorySlider の window キーハンドラ修正

### P1 — IA と体験の骨格(改善の本丸)
6. トップページ再構成: Coming Soon 撤去/圧縮、「解釈」コンテンツ(記事+予測+銘柄の見どころ)を主役に再配置。ナビから `/#featured` を外す
7. 予測の track record UI(的中率サマリ、答え合わせアーカイブ、確率推移チャート)— 戦略の濠を UI に実装
8. フォーカスリング体系の導入(`--ring` トークン + `focus-visible:` 一括適用)+ reduced-motion 対応
9. 銘柄一覧: 行クリック化、フィルタ即時反映、モバイル列の再設計(前日比優先)
10. sitemap.ts 実装 + structuredData.ts の配線(銘柄=Corporation、記事=NewsArticle、パンくず)
11. 巨大ファイル分割 + `/stocks/7203` 逆転構造の正規化(以降の改善速度のため)

### P2 — 差別化の上積み
12. ダークモード(トークン基盤は準備済み)
13. 記事セリフ体の正式フォント採用(Noto Serif JP)
14. インタラクティブチャート(期間切替・ホバー値表示)
15. 業界ハブページ(レイヤー②の受け皿)
16. 検索の横断化(記事・業界・テーマを含める)+「すべての結果」ページ
